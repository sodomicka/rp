# Garde_Cap - Passe 2 Shinbushiki

- version : W2
- dossier : Roadmap/Shinbushiki (non indexe au Sommaire, comme les roadmaps)
- nature : document de PILOTAGE inter-threads. Source de build uniquement, jamais fetche en narration.
- role : garder le cap entre les threads de Passe 2 (une roadmap par thread). Charge en OUVERTURE de chaque thread de build, mis a jour en CLOTURE.

## Protocole d'un thread de Passe 2
1. Ouverture : BIBLE en projet + ce fichier fetche + roadmap precedente si utile. Fetch selectif des fiches convoquees par l'arc.
2. Boucle serree sur LA roadmap du thread : grandes lignes -> confrontation au canon -> fiches creees/enrichies A CHAUD -> peaufinage -> generation ASCII.
3. Propagation a chaud : patches de fiches livres dans le meme thread. BIBLE differee (backlog B3 dans _Implications).
4. Cloture : mettre a jour ce fichier (statuts, dispatch, versions), annoncer la roadmap du thread suivant.

## Conventions actees
- Chronologie : marquage de Neji = an 0. Reperes derives : cf. Roadmap_00.
- Perimetre : R00 (flash-backs) + marquage -> veille de l'ere Boruto. Ere Boruto hors build. Canon manga only (hors films et fillers).
- Format : riche (a la Frieren), pas la table v8.2. ASCII strict, verif grep avant sortie.
- Granularite : 1 etape = 1 arc = 1 fiche d'arc (Passe 3).
- Double fil dans chaque roadmap : rail canon vecu par le vaisseau + fil souterrain du parasite (paliers d'extraction, refrenement, veille Isshiki, semis d'attachement, Zetsu).
- R00 : reference non jouable, produira des fiches d'arc marquees flashback.
